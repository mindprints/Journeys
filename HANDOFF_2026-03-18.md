## Known Issues / Watch Points
1. **`google/gemini-3.1-flash-image-preview` response format** — the Gemini image model returns a base64 data URL in `choices[0].message.images[0].image_url.url`. If OpenRouter changes this shape or the model is unavailable, the server logs `[generate-image] no image in response:` with the raw payload for diagnosis.
2. **`.env` vs `.env.example`** — user has `OPENROUTER_IMAGE_MODEL=google/gemini-3.1-flash-image-preview` set in their live `.env`. The `.env.example` now matches.
3. **Wikipedia normal path images** — the `else` branch in Step 3 calls `generate_ai_image()` synchronously during poster generation; if Gemini is slow or rate-limited this will add ~10–30s per topic with no thumbnail. Consider a timeout or making it opt-in via a flag if batch runs become sluggish.
